# 2026-04-06 工作日志

## 彻底修复空白页（根本原因：location.hash 双 # 问题）

- **根本原因**：`window.location.hash = '#home'` 这种写法会导致 URL 变成 `...##home`（双 #），因为 `location.hash` 赋值时浏览器会自动加 `#` 前缀，若传入值本身带 `#` 则重复
- **影响范围**：`bindNavEvents()` 中导航点击、`router.navigate()` 中路由跳转、Logo 点击等所有设置 hash 的地方
- **修复方案**：赋值前统一用 `.startsWith('#') ? href.slice(1) : href` 去掉 `#` 再赋值
- **附加改进**：移除所有内联 `onclick="router.navigate(...)"` 改为依赖 `href` 自然触发 hashchange；给 `renderHome` 和 `initApp` 加了 try-catch，异常有提示而非静默空白
- **commit**: `41d26da`

## 修复 router.navigate() 双 # 问题

- **问题**：`router.navigate('#articles')` 会将 `window.location.hash` 设为 `#articles`，导致 URL 变成 `...##articles`（双 #）
- **根因**：`window.location.hash = path` 中，浏览器会自动在 path 前加 `#`，如果 path 本身已包含 `#`，就会出现双 #
- **修复**：在 `navigate()` 方法中，若 path 以 `#` 开头则 `slice(1)` 后再赋值
- **修改文件**：`js/router.js`
- **已推送**：commit `b2dad1d`
